## Reference Configuration

1. `vite.config.js` must configure the `base` path to support sub-path deployment, support file imports via `@`, and do not configure a port number.

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  base: './',
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})

```

2. Router configuration must use hash mode to avoid server-side routing configuration issues.

```javascript
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    // route configurations
  ]
})

```
